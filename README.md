# iUnitRadar v1 — Калькулятор Unit-экономики B2B SaaS

🔗 **Live Demo:** [https://addito-5g.github.io/iUnitRadar_v1/](https://addito-5g.github.io/iUnitRadar_v1/)

Инструмент для расчёта ключевых метрик (MRR, LTV:CAC, NRR, Churn, ROMI) с возможностью сохранения и шеринга сценариев через ссылку.

## 🚀 Особенности
*   **Полный цикл метрик:** От MRR/ARR до Health Score и Rule of 40.
*   **Remote Sharing:** Генерация уникальных ссылок (`?calc=<uuid>`) для отправки расчётов коллегам или клиентам. Хранение снепшотов в Supabase.
*   **Чистая архитектура:** Рефакторинг legacy-кода в модульную структуру на ES Modules без сборщиков. Разделение логики, UI и состояния.
*   **Надёжность:** Обработка ошибок сети, валидация данных и версионирование схемы.

## 🛠 Технологии
*   **Frontend:** Vanilla JS (ES Modules), CSS.
*   **Backend:** Supabase (PostgreSQL + JSONB) через прямой REST API.
*   **Deploy:** GitHub Pages (статический хостинг).

## ⚙️ Настройка

1.  Создайте проект в [Supabase](https://supabase.com/) и выполните скрипт из `supabase/schema.sql`.
2.  Заполните `config.js` своими ключами:
    ```javascript
    window.__APP_CONFIG__ = {
      supabaseUrl: "https://YOUR_PROJECT.supabase.co",
      supabaseAnonKey: "YOUR_SUPABASE_ANON_KEY"
    };
    ```
3.  Запустите локально: `npx serve .` или `python -m http.server 8080`.

## 📂 Структура проекта
Проект разделён на слои для удобства поддержки:
*   `src/lib/` — чистые функции расчётов, валидации и форматирования.
*   `src/state/` — управление состоянием приложения и localStorage.
*   `src/features/` — UI-компоненты калькулятора и логика шеринга.

## 📝 Примечание
Исходный код оригинального продукта был недоступен. Данная версия — осознанная пересборка с сохранением бизнес-логики и добавлением функционала удалённого доступа. Для чувствительных данных рекомендуется настроить приватный доступ к снепшотам.
